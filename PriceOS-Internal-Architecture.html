1 Executive Overview

PriceOS is an AI-powered dynamic pricing copilot purpose-built for Dubai short-term rental property managers. It replaces manual pricing spreadsheets with an autonomous multi-agent system that scans the internet for market intelligence, benchmarks competitors in real-time, and generates daily pricing recommendations — all through a conversational interface called Aria.

💡 What Makes PriceOS Unique

Unlike traditional pricing tools that use static rules, PriceOS deploys 10 specialized AI agents that collaborate in real-time — scanning for geopolitical threats, competitor prices, events, and economic signals to generate defensible pricing decisions.

Capability Traditional Tools PriceOS
Pricing decisions Manual / rule-based AI agents with real-time internet access
Market intelligence Static quarterly reports Live scanning (wars, events, currency, weather)
User interface Dashboards with sliders Conversational AI ("Aria, what should I price?")
Competitor data Manual surveys Real-time Airbnb/Booking.com scraping
Safety guardrails None 4-layer protection: PriceGuard + Guardrails Agent

3 System Architecture

3.1 Two-Phase Architecture

PriceOS operates in two distinct phases. This is the most important architectural concept:

⚡ Phase 1: Setup

Trigger: User clicks "Run Aria"
Duration: 15-30 seconds
Agents: 6 (Marketing) + 7 (Benchmark) + 10 (Guardrails)
Internet: ✅ Yes — scans the web

💬 Phase 2: Chat

Trigger: User sends a message
Duration: 5-15 seconds
Agents: 1 (Aria) → 2, 3, 4, 5
Internet: ❌ No — uses cached data

🔑 Key Insight

Only Agents 6 and 7 have internet access. All chat-phase agents work with data cached from the last "Run Aria" scan. If an event is announced between scans, agents will be blind to it until the user triggers a new scan.

6 Safety & Anti-Hallucination

PriceOS implements 4 layers of protection against AI errors:

Layer Agent Protection Example
Layer 1 Agent 6 "2026 Mention Rule" — rejects events without explicit year on source Prevents 2025 Art Dubai dates being used for 2026
Layer 2 Agent 7 Bedroom-aware rate thresholds (1BR <1500, 4BR <6000 AED) Rejects monthly rental data mistaken as nightly
Layer 3 Agent 5 Geopolitical crisis override — wars cancel event premiums Iran tensions → drop to P25 pricing for occupancy
Layer 4 Agent 1 Anomaly alert — warns user if market data seems wrong "Market median seems wrong — using conservative pricing"

7 The 7-Step Intelligence Sweep

Agent 6 (Marketing) executes this comprehensive sweep every time "Run Aria" is triggered:

  • 🔴 Geopolitical Threat Scan — Travel advisories, Iran-UAE tensions, Yemen Houthi attacks, flight disruptions, pandemics. Impact: -40% to -60%.
  • 📅 Calendar & Religious Dates — Ramadan phase (early/mid/late/Eid), UAE holidays, school holidays. Impact: -20% to +50%.
  • 🎪 Major Events — Exhibitions, conferences, festivals, concerts, sports in date range. Impact: +10% to +30%.
  • 🏘️ Neighborhood Intelligence — Landmark closures, local activities, micro-events. Impact: -10%.
  • 💹 Economic & Currency Signals — INR/GBP/EUR exchange rates, oil prices, hotel occupancy, tourism stats. Impact: -3% to +5%.
  • 🌡️ Weather & Environment — Heat waves, sandstorms, pleasant conditions. Impact: -40% (summer).
  • 📱 Viral & Trending — Dubai trending on social media, scam warnings, new attractions. Impact: ±5%.

13 Key Design Decisions

Why Multi-Agent (Not Single LLM)?

A single LLM cannot search the internet, validate prices, AND generate reports in one call. 10 specialized agents each have a narrow, verifiable scope — if PriceGuard rejects a price, we know exactly why.

Why Two-Phase (Not Real-Time)?

Internet searches take 15-30 seconds — too slow for real-time. Competitor prices don't change minute-by-minute. Property managers want to review pricing before applying.

Why Floor/Ceiling Guardrails?

Pricing errors are asymmetric: overpricing loses one night; underpricing creates a viral "cheap" perception that's hard to undo. Guardrails create a safe zone for AI decisions.

15 Detailed Agent Playbook

This section provides a deep-dive into the logic and output format of each core agent in the PriceOS ecosystem.

Agent 1 CRO Router (Aria)

Responsibility: The "Brain" of PriceOS. Aria is an orchestrator that translates loose human intentions into structured multi-agent tasks.

How it Works:
  1. Parses the user message (e.g., "Full analysis for next week").
  2. Constructs the System Payload (metrics, bookings, market data).
  3. Parallel-executes Agents 2, 3, 4, 5.
  4. Merges JSON results into a unified XML-structured report.
Sample Output (Internal Report State):
<analysis_report>
  <executive_summary>🔴 Red Alert: Airspace closure NOTAM issued...</executive_summary>
  <opportunity_score>85/100</opportunity_score>
  <sub_reports>
    <property_logic>Gap nights detected on Mar 12-14...</property_logic>>
    <price_results>[14 proposals generated]</price_results>
  </sub_reports>
</analysis_report>

Agent 2 Property Analyst

Responsibility: Maximizing occupancy through gap detection and inventory management.

How it Works:

Scans the inventory array day-by-day. If it finds isolated available nights between bookings (e.g., a 2-night gap between two 5-night bookings), it flags them for special pricing or reduced Min Stay logic.

Sample Output:
{
  "gaps": [{"start": "2026-03-12", "end": "2026-03-14", "nights": 2}],
  "restrictions": "Reduce Min Stay from 3 to 2 for Mar 12 gap",
  "seasonal_analysis": "Entry into Ramadan shoulder period: -10% ADR expected."
}

Agent 5 PriceGuard

Responsibility: The Final Validator. It holds "Veto Power" over all pricing decisions.

How it Works:
  1. Receives suggested premiums from Market Research (Agent 4).
  2. Computes Final_Price = (Base_Rate * Trend_Multiplier) + Context_Premiums.
  3. Force-clamps the result to the floor_price and ceiling_price.
  4. Validates the price against Agent 7's P50/P90 benchmarks.
Sample Output (Individual Proposal):
{
  "date": "2026-03-15",
  "proposedPrice": 850,
  "reasoning": {
    "market": "P75 position maintained despite Ramadan mid-phase",
    "occupancy": "Property at 0% occupancy for window; price lowered to floor",
    "threats": "Geopolitical risk level 3 active; -15% safety buffer applied"
  }
}

Agent 6 Marketing Intelligence

Responsibility: Real-time internet scanning with a "Skepticism Layer."

How it Works:

Uses Perplexity Sonar to execute a 7-step sweep. Every "alert" found is fact-checked: Does it mention 2026? Is there a second official source? This prevents the system from over-reacting to fake or outdated news.

Sample Output:
{
  "news": [{
    "headline": "Dubai Airspace Remains Open (Official WAM)",
    "category": "security",
    "sentiment": "positive",
    "confidence": 98,
    "description": "Cross-verified with Emirates and DXB Airport arrivals."
  }],
  "demand_outlook": {"trend": "strong", "reason": "Art Dubai surge"}
}

Agent 3 Booking Intelligence

Responsibility: Analyzing internal booking "pacing" and velocity to detect demand strength before the market does.

How it Works:

Compares recent_reservations against historical averages for the same period. If booking lead time is shrinking, it flags a "High Velocity" signal to raise prices.

Sample Output:
{
  "velocity_score": 8.5,
  "pacing_verdict": "Ahead of target (+12% YoY)",
  "channel_mix": "Direct: 15%, Airbnb: 70%, Booking: 15%"
}

Agent 4 Market Research

Responsibility: Translating raw external events into numerical price multipliers.

How it Works:

Reviews the market_events and news tables. It assigns an impact_multiplier to each date based on proximity to events like Art Dubai or COP28.

Sample Output:
{
  "event_scores": [{"event": "Art Dubai", "score": 1.25, "dates": ["2026-03-06", "2026-03-09"]}],
  "sentiment_multiplier": 0.95
}

Agent 7 Benchmark Agent

Responsibility: Continuous competitor "scraping" and price normalization.

How it Works:

Executes live searches for "Dubai [Area] 1BR Apartment" on Airbnb/Booking. It extracts the exact nightly rate for the next 14 days, excluding cleaning fees, to ensure an apples-to-apples comparison.

Sample Output:
{
  "verdict": "Priced Aggressively (P90)",
  "p50_median": 650,
  "p90_high": 1200,
  "comps": [{"name": "Palm Villa 4", "price": 1150, "rating": 4.9, "reviews": 122}]
}

Agent 10 Guardrails Agent

Responsibility: Establishing the "Safe Zone" for AI operations.

How it Works:

Runs only during the Setup Phase. It looks at the property's fixed costs and the Market Median (P50) to set a Floor below which the AI cannot go, and a Ceiling that prevents price gouging.

Sample Output:
{
  "new_floor_price": 450,
  "new_ceiling_price": 1850,
  "reasoning": "Floor set at 15% below P50 to ensure high occupancy in low season."
}

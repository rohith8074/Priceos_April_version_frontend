What triggers this phase?

The "Run Aria" button in the UI. It calls /api/market-setup which runs Agent 6 and Agent 7 in parallel using Perplexity Sonar (internet search LLM). Their results are saved to the database — not sent to the chat. The chat agents read from these tables later.

Defense in Depth — Why Two Layers?

PriceGuard is an LLM — it usually follows instructions but can occasionally hallucinate or skip steps. The server-side enforceGuardrails() is hardcoded TypeScript — it always runs, cannot be bypassed, and guarantees business rules are upheld even if every prompt is deleted.

All 9 Agents — Responsibilities & Data

Chat Agent

🤖 Agent 1: Aria (CRO Router)

gemini-3.0-flash-preview · temp 0.2 · 4000 tokens
User-facing orchestrator. Classifies intent, delegates to sub-agents, merges outputs into a structured 10-section analysis.
Responsibilities
  • Classify user intent from natural language
  • Route to correct sub-agents using the routing table
  • Merge sub-agent outputs into a unified response
  • Classify dates: Protected / Healthy / At Risk / Distressed
  • Generate 10-section Full Analysis report
  • Never compute pricing — always delegate to PriceGuard
📥 Receives (injected once)
analysis_window property metrics benchmark market_events recent_reservations
📤 Returns
routing{} proposals[] chat_response
Chat Agent

📊 Agent 2: Property Analyst

gpt-4o-mini · temp 0.1 · 1500 tokens
Calendar and gap analysis specialist. Finds vacant gaps, recommends LOS changes, projects revenue.
Responsibilities
  • Detect gap types: 1-night orphan, 2-night micro, 3-night, last-minute
  • Recommend LOS relaxation (prefer over discounting)
  • Auto-revert: if LOS change unfilled 48hrs → revert
  • Identify seasonal patterns
  • Calculate confirmed / potential / projected revenue
  • Never suggest below floor price
📤 Returns
gap_nights[] restrictions[] seasonal{} revenue{}
Chat Agent

📈 Agent 3: Booking Intelligence

gpt-4o-mini · temp 0.1 · 1500 tokens
Reservation pattern analyzer. Extracts booking velocity, LOS trends, channel mix, and cancellation signals.
Responsibilities
  • Calculate booking velocity (accelerating/stable/decelerating)
  • Analyze length of stay buckets (1-2, 3-4, 5-7, 7+)
  • Channel mix breakdown (Airbnb, Booking.com, Direct %)
  • Day-of-week fill patterns
  • Cancellation rate analysis
  • Lead time analysis (how far in advance guests book)
📤 Returns
velocity{} los_buckets{} channel_mix{} revenue_analysis{}
Chat Agent

🏆 Agent 4: Market Research

gpt-4o-mini · temp 0.1 · 1500 tokens
Reads pre-cached market intelligence (from Agents 6+7). Returns event impact, competitor positioning, and pricing recommendations.
Responsibilities
  • Parse and structure event data with impact levels
  • Calculate market position (percentile vs P25/P50/P75/P90)
  • Assign event premium factors (low=1.05x, medium=1.15x, high=1.30x)
  • Return competitor comparisons
  • Determine verdict: UNDERPRICED / FAIR / PREMIUM
📤 Returns
events[] positioning{} competitors[] recommendations{}
Safety Gate

🛡️ Agent 5: PriceGuard

gpt-4o-mini · temp 0.0 · 1200 tokens
Pricing engine + final safety validator. Has unconditional veto power — CRO Router cannot override a REJECTED verdict.
Responsibilities
  • STEP 1: Compute — base × event_factor × occupancy_adj
  • STEP 2: Clamp to floor/ceiling (NO EXCEPTIONS)
  • STEP 3: Validate — reject if |change| > 50%
  • STEP 4: Assign risk — <5% Low, 5-15% Medium, >15% High
  • Differentiate: weekday ≠ weekend ≠ event prices
  • Fallback: if benchmark = 0, use MAX(current, floor)
📤 Returns (per date)
proposed_price change_pct risk_level guard_verdict reasoning

Key Principle: Chat Agents Have ZERO Database Access

Agents 1-5 (Aria, Property Analyst, Booking Intel, Market Research, PriceGuard) cannot touch the database. All data is injected into the first prompt message as a JSON payload by route.ts. This makes the system secure, testable, and deterministic — the agents work only with what they're given.
